Chains & LCEL

LangChain Expression Language (LCEL) is the modern, composable way to build chains using the | pipe operator. Every component — prompts, models, parsers, retrievers — is a Runnable that can be chained, parallelised, and streamed.

What is a Runnable?

Every LCEL component implements the Runnable interface, which provides four core methods:

MethodDescription
.invoke(input)Run synchronously, return result
.stream(input)Return an iterator of streamed chunks
.batch([inputs])Run multiple inputs in parallel
.ainvoke / .astream / .abatchAsync variants of the above

Basic chain with | pipe operator

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# Each | connects Runnables: prompt → model → parser
chain = (
    ChatPromptTemplate.from_template("Summarise this in one sentence: {text}")
    | llm
    | StrOutputParser()
)

result = chain.invoke({"text": "Docker is an open platform for developing, shipping, and running applications..."})
print(result)   # "Docker is a platform for containerising and running applications."

Sequential chain — multi-step pipeline

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

step1_prompt = ChatPromptTemplate.from_template(
    "Extract the key technical concepts from: {text}"
)
step2_prompt = ChatPromptTemplate.from_template(
    "For each concept: {concepts}\n\nWrite a one-line definition."
)

step1 = step1_prompt | llm | StrOutputParser()
step2 = step2_prompt | llm | StrOutputParser()

# Chain the chains: output of step1 becomes {concepts} for step2
pipeline = step1 | (lambda concepts: {"concepts": concepts}) | step2

result = pipeline.invoke({"text": "Kubernetes uses Pods, Services, and Deployments to run containers at scale."})
print(result)

RunnableParallel — run steps side-by-side

from langchain_core.runnables import RunnableParallel

# Run two LLM calls simultaneously
parallel_chain = RunnableParallel(
    summary=ChatPromptTemplate.from_template("Summarise: {topic}") | llm | StrOutputParser(),
    example=ChatPromptTemplate.from_template("Give one example of: {topic}") | llm | StrOutputParser(),
)

result = parallel_chain.invoke({"topic": "Docker volumes"})
print(result["summary"])   # "Docker volumes persist container data..."
print(result["example"])   # "Example: -v /host/data:/app/data"

RunnablePassthrough — pass input through

from langchain_core.runnables import RunnableParallel, RunnablePassthrough

# Keep the original input alongside a computed value
chain = RunnableParallel(
    question=RunnablePassthrough(),          # pass {question} as-is
    answer=(
        ChatPromptTemplate.from_template("Answer briefly: {question}")
        | llm | StrOutputParser()
    ),
)

result = chain.invoke({"question": "What is a Docker image?"})
print(result["question"])   # "What is a Docker image?"
print(result["answer"])     # "A Docker image is a read-only template..."

RunnableLambda — custom Python functions

from langchain_core.runnables import RunnableLambda

def word_count(text: str) -> dict:
    return {"text": text, "words": len(text.split())}

# Wrap any Python function as a Runnable
chain = (
    ChatPromptTemplate.from_template("Write a poem about {topic}")
    | llm
    | StrOutputParser()
    | RunnableLambda(word_count)
)

result = chain.invoke({"topic": "containers"})
print(result["words"])   # e.g. 42

Streaming a chain

chain = (
    ChatPromptTemplate.from_template("Write a short story about {topic}")
    | ChatOpenAI(model="gpt-4o-mini", streaming=True)
    | StrOutputParser()
)

# Print tokens as they stream in
for chunk in chain.stream({"topic": "a Kubernetes pod that escapes its namespace"}):
    print(chunk, end="", flush=True)

# Async version
import asyncio
async def stream():
    async for chunk in chain.astream({"topic": "a Docker container at sea"}):
        print(chunk, end="", flush=True)

asyncio.run(stream())

Branching with RunnableBranch

from langchain_core.runnables import RunnableBranch

code_prompt  = ChatPromptTemplate.from_template("Write Python code for: {task}")
prose_prompt = ChatPromptTemplate.from_template("Explain in plain English: {task}")

branch = RunnableBranch(
    (lambda x: "code" in x["task"].lower(),  code_prompt  | llm | StrOutputParser()),
    (lambda x: True,                          prose_prompt | llm | StrOutputParser()),  # default
)

print(branch.invoke({"task": "Write code to list Docker containers"}))
print(branch.invoke({"task": "Explain what Kubernetes does"}))

Error handling with fallbacks

from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

primary_llm  = ChatOpenAI(model="gpt-4o")
fallback_llm = ChatAnthropic(model="claude-haiku-4-5")

# If gpt-4o throws (rate limit, timeout), automatically retry with Claude
resilient_llm = primary_llm.with_fallbacks([fallback_llm])

chain = (
    ChatPromptTemplate.from_template("Answer: {question}")
    | resilient_llm
    | StrOutputParser()
)

result = chain.invoke({"question": "What is RAG?"})
LCEL vs legacy chains: If you see LLMChain, SequentialChain, or ConversationChain in older code, these are deprecated. Replace them with LCEL (prompt | llm | parser). LCEL chains are simpler, faster, and support streaming + async natively.
