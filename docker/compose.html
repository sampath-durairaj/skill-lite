Docker Compose

Docker Compose lets you define and run multi-container applications with a single YAML file. Instead of running long docker run commands manually, you describe your entire stack — services, networks, volumes — and bring it all up with docker compose up.

docker-compose.yml structure

version: "3.9"   # Compose file format version

services:        # One entry per container
  web:
    ...
  db:
    ...

networks:        # Optional custom networks
  appnet:

volumes:         # Optional named volumes
  pgdata:

A complete example

version: "3.9"

services:
  web:
    build: .                      # Build from Dockerfile in current dir
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://admin:secret@db:5432/mydb
      NODE_ENV: production
    depends_on:
      db:
        condition: service_healthy  # Wait for db healthcheck
    networks:
      - appnet
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: mydb
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - appnet

  redis:
    image: redis:7-alpine
    networks:
      - appnet

networks:
  appnet:

volumes:
  pgdata:

Compose CLI commands

# Start all services (build if needed), detached
docker compose up -d

# Build/rebuild images before starting
docker compose up -d --build

# Stop and remove containers (preserves volumes)
docker compose down

# Stop and remove containers AND volumes
docker compose down -v

# View logs for all services
docker compose logs -f

# View logs for one service
docker compose logs -f web

# List running services
docker compose ps

# Execute a command in a running service container
docker compose exec web bash
docker compose exec db psql -U admin mydb

# Run a one-off command in a new container (then remove it)
docker compose run --rm web npm run migrate

# Scale a service (Compose v2 syntax)
docker compose up -d --scale web=3

# Pull the latest images defined in compose file
docker compose pull

# Validate compose file syntax
docker compose config

Environment variable substitution

Compose automatically reads a .env file in the project directory and substitutes ${VARIABLE} references in the YAML.

# .env file
POSTGRES_PASSWORD=supersecret
APP_PORT=3000
services:
  web:
    ports:
      - "${APP_PORT}:3000"
  db:
    environment:
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
Best practice: commit a .env.example with placeholder values and gitignore the real .env. This documents required variables without exposing secrets.

Compose profiles

Profiles let you selectively start services. Services without a profiles key always start; services with one only start when that profile is activated.

services:
  web:
    image: myapp        # always starts

  debug-tools:
    image: busybox
    profiles: [debug]   # only with --profile debug
# Start with the debug profile
docker compose --profile debug up -d

Overriding with multiple files

# Base file for all environments
docker compose -f docker-compose.yml up

# Override for development (merges on top of base)
docker compose -f docker-compose.yml -f docker-compose.dev.yml up

# Override for CI
docker compose -f docker-compose.yml -f docker-compose.ci.yml up
