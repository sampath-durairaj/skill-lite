Containers

A container is a running instance of an image — an isolated process with its own filesystem, network, and process tree, but sharing the host OS kernel. Understanding the container lifecycle is essential to working with Docker day-to-day.

Container lifecycle

created

  • docker create
  • Filesystem ready
  • Not yet running

running

  • docker start / run
  • Process active
  • Using CPU & memory

paused

  • docker pause
  • Frozen in memory
  • No CPU

stopped

  • Process exited
  • Filesystem preserved
  • Can be restarted

Stopped containers retain their writable layer. Use docker rm to delete a stopped container and reclaim disk space.

Running containers

# Run nginx detached, publish port 80 → localhost:8080
docker run -d -p 8080:80 --name web nginx

# Run an interactive Alpine shell (deleted on exit with --rm)
docker run --rm -it alpine sh

# Pass environment variables
docker run -d \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=secret \
  -e POSTGRES_DB=mydb \
  --name db postgres:16

# Run with a restart policy (restarts unless manually stopped)
docker run -d --restart unless-stopped --name app myimage

Managing running containers

# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Stop a container gracefully (sends SIGTERM, then SIGKILL after timeout)
docker stop web

# Force kill (SIGKILL immediately)
docker kill web

# Start a stopped container
docker start web

# Restart
docker restart web

# Pause / unpause (freeze/unfreeze process)
docker pause web
docker unpause web

# Remove a stopped container
docker rm web

# Remove a running container forcefully
docker rm -f web

# Remove all stopped containers
docker container prune

Debugging containers

# Stream container logs
docker logs -f web

# Show last 50 lines of logs
docker logs --tail 50 web

# Execute a command inside a running container
docker exec -it web bash
docker exec web cat /etc/nginx/nginx.conf

# Show live resource usage
docker stats

# Inspect full container metadata (JSON)
docker inspect web

# Show mapped ports
docker port web

# Show running processes
docker top web

# Copy files between host and container
docker cp web:/etc/nginx/nginx.conf ./nginx.conf
docker cp ./new-config.conf web:/etc/nginx/nginx.conf

Restart policies

PolicyBehaviour
noNever restart automatically (default).
alwaysAlways restart, even after docker stop. Starts on daemon boot.
unless-stoppedLike always but does not restart if manually stopped.
on-failure[:n]Restart only if the process exits with a non-zero code. Optionally limit retries.
Ephemeral writeable layer: any file written inside a container (outside a mounted volume) lives only as long as the container. When the container is removed, those writes are lost. Always use volumes for data that must outlast the container.
