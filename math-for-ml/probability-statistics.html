Probability & Statistics

Probability lets a model express uncertainty instead of a single guess; statistics lets you decide whether what you measured is real or just noise. Together they underpin loss functions, model evaluation, and every "how confident are we?" question in ML.

Random variables and distributions

A random variable maps outcomes to numbers. A distribution describes how likely each value is.

DistributionModelsML example
BernoulliA single yes/no outcomeBinary classification label
BinomialCount of successes in n trialsNumber of correct predictions out of N
Normal (Gaussian)Continuous, bell-shaped, symmetric around the meanWeight initialization, measurement noise
UniformEvery value in a range equally likelyRandom sampling, initial exploration
CategoricalOne of k discrete outcomesMulti-class softmax output
import numpy as np

rng = np.random.default_rng(42)

rng.normal(loc=0, scale=1, size=5)      # 5 samples from N(0, 1)
rng.uniform(0, 1, size=5)                 # 5 samples, uniform in [0, 1)
rng.binomial(n=10, p=0.5, size=5)          # 5 samples, "heads out of 10 flips"
rng.choice(["cat", "dog", "bird"], size=5, p=[0.5, 0.3, 0.2])   # categorical

Mean, variance, standard deviation

x = np.array([2, 4, 4, 4, 5, 5, 7, 9])

x.mean()          # 5.0   — the average
x.var()            # 4.0   — average squared distance from the mean
x.std()             # 2.0   — sqrt(variance), same units as the data

# Standardization — the single most common preprocessing step in ML
z = (x - x.mean()) / x.std()      # rescales to mean 0, std 1
Why standardize? Features on wildly different scales (e.g. "age" 0-100 vs "income" 0-200,000) make gradient descent slow and unstable. Standardizing every feature to mean 0, std 1 is standard practice before training most models.

Conditional probability and Bayes' theorem

# P(A|B) = P(B|A) * P(A) / P(B)
#
# Spam filter example:
# P(spam)         = 0.3     — prior: 30% of email is spam
# P("free"|spam)   = 0.8    — likelihood: 80% of spam contains "free"
# P("free")          = 0.35  — evidence: 35% of all email contains "free"

p_spam = 0.3
p_free_given_spam = 0.8
p_free = 0.35

p_spam_given_free = (p_free_given_spam * p_spam) / p_free
print(f"{p_spam_given_free:.3f}")   # 0.686 — posterior: P(spam | "free")

This is exactly how a Naive Bayes classifier works: combine a prior belief with new evidence to get an updated (posterior) probability.

Correlation and covariance

x = np.array([1, 2, 3, 4, 5])
y = np.array([2, 4, 5, 4, 5])

np.cov(x, y)                    # covariance matrix — direction of joint variation
np.corrcoef(x, y)                 # correlation matrix — normalized to [-1, 1]
# corrcoef[0,1] close to 1  -> strong positive relationship
# corrcoef[0,1] close to 0  -> little linear relationship
# corrcoef[0,1] close to -1 -> strong negative relationship

The Central Limit Theorem

The average of enough independent random samples tends toward a normal distribution, regardless of the original data's distribution. This is why so many statistical tests, confidence intervals, and evaluation metrics assume normality — averaging (over a batch, over cross-validation folds) makes that assumption reasonable even when individual data points aren't normal.

Statistics in model evaluation

from scipy import stats

# Is model A's accuracy significantly better than model B's, or just noise?
scores_a = [0.91, 0.89, 0.93, 0.90, 0.92]
scores_b = [0.87, 0.88, 0.85, 0.89, 0.86]

t_stat, p_value = stats.ttest_ind(scores_a, scores_b)
print(f"p-value: {p_value:.4f}")
# p < 0.05 conventionally suggests the difference is unlikely to be due to chance
