Linear Algebra

Linear algebra is the language of data in machine learning. A single training example is a vector, a batch of examples is a matrix, and a neural network layer is little more than a matrix multiplication followed by a nonlinearity. This page builds intuition using NumPy — the same structures shown here are what NumPy's ndarray was built to represent.

Scalars, vectors, matrices, tensors

TermShapeExampleIn ML
Scalar0-D5A single loss value, a learning rate
Vector1-D, (n,)[1, 2, 3]One data sample's features, a word embedding
Matrix2-D, (m, n)[[1,2],[3,4]]A batch of samples, a layer's weights

Vector operations

import numpy as np

v = np.array([1, 2, 3])
w = np.array([4, 5, 6])

v + w                     # [5 7 9]        — element-wise addition
v - w                      # [-3 -3 -3]
3 * v                       # [3 6 9]        — scalar multiplication

# Dot product — sum of element-wise products. Core of every layer.
np.dot(v, w)                 # 1*4 + 2*5 + 3*6 = 32
v @ w                          # same, using the @ operator

# Norm — the "length" of a vector (Euclidean / L2 norm)
np.linalg.norm(v)               # sqrt(1² + 2² + 3²) ≈ 3.742

# Cosine similarity — how "aligned" two vectors are (used in embeddings, RAG)
cos_sim = np.dot(v, w) / (np.linalg.norm(v) * np.linalg.norm(w))
Dot product intuition: The dot product measures how much two vectors point in the same direction — it's the mathematical basis for cosine similarity in embeddings, and it's literally what one neuron computes: weighted_sum = np.dot(inputs, weights).

Matrix operations

A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

A + B                # element-wise addition, same shape required
A * B                 # element-wise (Hadamard) product — NOT matrix multiplication

A @ B                  # matrix multiplication (or np.matmul(A, B))
# Rule: (m, k) @ (k, n) -> (m, n) — inner dimensions must match

A.T                     # transpose — flips rows and columns
np.eye(2)                # identity matrix — the "1" of matrix multiplication
np.linalg.inv(A)          # inverse — A @ inv(A) == identity (if A is invertible)
np.linalg.det(A)           # determinant — 0 means A has no inverse

Why matrix multiplication is everywhere in ML

# A dense (linear) layer for a batch of inputs
X = np.random.randn(32, 10)     # batch of 32 samples, 10 features each
W = np.random.randn(10, 5)       # weights: 10 inputs -> 5 outputs
b = np.random.randn(5)             # bias per output

Y = X @ W + b                        # shape (32, 5) — output for every sample, every unit
# This single line IS a fully-connected / linear / dense layer.

Stack several of these (each followed by a nonlinearity like ReLU or sigmoid) and you have a neural network. Every forward pass is a sequence of matrix multiplications.

Eigenvalues, eigenvectors & dimensionality reduction

A = np.array([[2, 0], [0, 3]])

eigenvalues, eigenvectors = np.linalg.eig(A)
# eigenvalues:  [2. 3.]
# eigenvectors: directions that A only scales, never rotates

# PCA (dimensionality reduction) finds the eigenvectors of the
# covariance matrix — the directions of greatest variance in the data.
data = np.random.randn(100, 5)
cov = np.cov(data.T)
eigvals, eigvecs = np.linalg.eig(cov)
top2 = eigvecs[:, np.argsort(eigvals)[::-1][:2]]   # top-2 principal components
reduced = data @ top2                                 # (100, 5) -> (100, 2)
