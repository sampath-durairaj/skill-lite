Calculus

Calculus is how models learn. Training a model means adjusting its parameters to minimize a loss function — and the direction that minimizes it fastest is given by the gradient, the multivariable generalization of a derivative. Every call to loss.backward() in PyTorch is calculus, done automatically.

Derivatives — the rate of change

The derivative of a function tells you its slope at a point: how much the output changes for a tiny change in input.

# f(x) = x²  →  f'(x) = 2x   (power rule: d/dx x^n = n*x^(n-1))

def f(x):
    return x ** 2

def numerical_derivative(f, x, h=1e-6):
    return (f(x + h) - f(x - h)) / (2 * h)     # slope of the secant line, h -> 0

numerical_derivative(f, 3.0)     # ≈ 6.0 — matches the analytical f'(x) = 2x at x=3

Partial derivatives and the gradient

A loss function depends on many parameters at once (every weight in the network). A partial derivative is the rate of change with respect to just one of them, holding the others fixed. The gradient stacks all partial derivatives into a vector that points in the direction of steepest increase.

# f(w1, w2) = w1² + w2²
# ∂f/∂w1 = 2*w1     ∂f/∂w2 = 2*w2
# gradient = [2*w1, 2*w2]

import numpy as np

def f(w):
    return w[0]**2 + w[1]**2

def gradient(f, w, h=1e-6):
    grad = np.zeros_like(w)
    for i in range(len(w)):
        w_plus, w_minus = w.copy(), w.copy()
        w_plus[i] += h
        w_minus[i] -= h
        grad[i] = (f(w_plus) - f(w_minus)) / (2 * h)
    return grad

w = np.array([3.0, 4.0])
gradient(f, w)     # ≈ [6.0, 8.0] — matches [2*3, 2*4]

The chain rule — how backpropagation works

Neural networks are functions composed of functions (layer after layer). The chain rule lets you compute the derivative of a composition by multiplying the derivatives of each piece — this is exactly what backpropagation automates.

# y = f(g(x))  →  dy/dx = f'(g(x)) * g'(x)
#
# A 2-layer network:  z = w*x + b,  y = sigmoid(z),  loss = (y - target)²
# dloss/dw = dloss/dy * dy/dz * dz/dw    ← chain rule, three local derivatives multiplied

def sigmoid(z): return 1 / (1 + np.exp(-z))

x, w, b, target = 2.0, 0.5, 0.1, 1.0

z = w * x + b
y = sigmoid(z)
loss = (y - target) ** 2

dloss_dy = 2 * (y - target)
dy_dz    = y * (1 - y)              # sigmoid derivative
dz_dw    = x                          # dz/dw = x since z = w*x + b

dloss_dw = dloss_dy * dy_dz * dz_dw     # chain rule
print(f"dloss/dw = {dloss_dw:.4f}")

Gradient descent — how learning happens

w = np.array([3.0, 4.0])
lr = 0.1

for step in range(20):
    grad = gradient(f, w)          # direction of steepest ascent
    w = w - lr * grad                # step downhill (minimize f)

print(w)     # converges toward [0, 0] — the minimum of f(w) = w1² + w2²
This is the entire training loop: compute the gradient of the loss with respect to every weight, then nudge each weight a small step in the opposite direction. Frameworks like PyTorch (autograd) and TensorFlow compute these gradients automatically via the chain rule — but the update rule is exactly the loop above.

Integrals — the other half of calculus

Where a derivative measures instantaneous rate of change, an integral accumulates a quantity over a range. In ML, integrals appear less often in day-to-day code, but underpin probability (the area under a probability density function must integrate to 1) and metrics like AUC (area under the ROC curve).

from scipy import integrate

# Area under a normal distribution's PDF between -1 and 1 (~68%, the "1-sigma" range)
from scipy.stats import norm
area, _ = integrate.quad(norm.pdf, -1, 1)
print(f"{area:.3f}")   # ≈ 0.683
