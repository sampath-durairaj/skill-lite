Matplotlib

Matplotlib is Python's foundational plotting library — it renders NumPy arrays and Pandas Series/DataFrames as line charts, bar charts, histograms, and scatter plots. Most other Python visualization libraries (Seaborn, Pandas' own .plot()) are built on top of it.

pip install matplotlib

Figure and Axes

Every Matplotlib plot has a Figure (the whole canvas) containing one or more Axes (an individual plot area with its own x/y axis). The object-oriented API (fig, ax = plt.subplots()) is preferred for anything beyond a quick one-off plot — it gives you explicit control and scales cleanly to multiple subplots.

import matplotlib.pyplot as plt
import numpy as np

x = np.linspace(0, 10, 100)
y = np.sin(x)

# Object-oriented API (recommended)
fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(x, y)
ax.set_title("Sine wave")
ax.set_xlabel("x")
ax.set_ylabel("sin(x)")
plt.show()

# Quick pyplot API (fine for one-off scripts)
plt.plot(x, y)
plt.title("Sine wave")
plt.show()

Common plot types

FunctionUse for
ax.plot(x, y)Line chart — trends over a continuous axis (time series)
ax.scatter(x, y)Relationship between two numeric variables
ax.bar(labels, values)Comparing values across categories
ax.barh(labels, values)Same, horizontal — long category labels
ax.hist(data, bins=20)Distribution of a single numeric variable
ax.boxplot(data)Median, quartiles, and outliers
ax.pie(values)Parts of a whole (use sparingly)
fig, ax = plt.subplots()
categories = ["Python", "NumPy", "Pandas", "Matplotlib"]
scores = [92, 85, 88, 79]
ax.bar(categories, scores, color="#3776ab")
ax.set_ylabel("Score")
ax.set_title("Library familiarity")
plt.show()

Multiple subplots

fig, axes = plt.subplots(1, 2, figsize=(10, 4))   # 1 row, 2 columns

axes[0].plot(x, np.sin(x))
axes[0].set_title("sin(x)")

axes[1].plot(x, np.cos(x), color="orange")
axes[1].set_title("cos(x)")

fig.tight_layout()      # prevent overlapping labels
plt.show()

# Grid of subplots
fig, axes = plt.subplots(2, 2, figsize=(8, 6))
axes[0, 0].plot(x, y)          # top-left
axes[1, 1].hist(np.random.randn(1000))  # bottom-right

Customization

fig, ax = plt.subplots()

ax.plot(x, np.sin(x), label="sin(x)", color="#3776ab", linewidth=2, linestyle="--")
ax.plot(x, np.cos(x), label="cos(x)", color="#f59e0b", linewidth=2)

ax.set_title("Trig functions", fontsize=14, fontweight="bold")
ax.set_xlabel("x")
ax.set_ylabel("y")
ax.legend()                  # show labels from each plot() call
ax.grid(True, alpha=0.3)      # light gridlines
ax.set_xlim(0, 10)              # x-axis range
ax.set_ylim(-1.5, 1.5)

Plotting directly from Pandas

import pandas as pd

df = pd.DataFrame({
    "month": ["Jan", "Feb", "Mar", "Apr"],
    "revenue": [1200, 1500, 1100, 1800],
})

df.plot(x="month", y="revenue", kind="bar", legend=False)   # uses Matplotlib under the hood
plt.ylabel("Revenue ($)")
plt.title("Monthly revenue")
plt.show()

Saving figures

fig.savefig("chart.png", dpi=200, bbox_inches="tight")
fig.savefig("chart.svg")     # vector format — scales without pixelation
fig.savefig("chart.pdf")
pyplot vs object-oriented API: plt.plot(...) implicitly acts on "the current figure," which gets confusing with multiple plots. Once you need more than a single simple chart, switch to fig, ax = plt.subplots() and call methods on ax directly — it's explicit and avoids state-related bugs.
