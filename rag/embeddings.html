Embeddings

Embeddings convert text into dense numeric vectors (arrays of floats) that capture semantic meaning. Texts with similar meaning cluster near each other in vector space — enabling similarity search without exact keyword matching. The embedding model you choose directly determines retrieval quality.

How embeddings work

Input text

  • "Docker containers are isolated"
  • "Pods run in Kubernetes"
  • "Weather is sunny today"

Embedding model

  • Neural network encoder
  • Maps tokens → vector
  • 1536 / 768 dimensions

Vector space

  • Docker ≈ Kubernetes (close)
  • Weather ≠ Docker (far)
  • Cosine similarity score

OpenAI Embeddings

from langchain_openai import OpenAIEmbeddings

# text-embedding-3-small — best price/performance
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",   # 1536 dims, ~$0.02/1M tokens
    dimensions=512,                    # reduce dims to save storage/cost
)

# Embed a single query
vector = embeddings.embed_query("What is a Docker container?")
print(f"Dimensions: {len(vector)}")        # 512
print(f"First values: {vector[:5]}")       # [0.023, -0.041, ...]

# Embed a batch of documents (more efficient)
texts = [
    "Docker runs containers",
    "Kubernetes orchestrates pods",
    "RAG retrieves documents",
]
vectors = embeddings.embed_documents(texts)
print(f"Embedded {len(vectors)} docs, each with {len(vectors[0])} dims")

HuggingFace — free, local models

pip install sentence-transformers langchain-huggingface
from langchain_huggingface import HuggingFaceEmbeddings

# Runs entirely locally — no API key, no cost
embeddings = HuggingFaceEmbeddings(
    model_name="sentence-transformers/all-MiniLM-L6-v2",  # 384 dims, fast
    model_kwargs={"device": "cpu"},      # or "cuda" for GPU
    encode_kwargs={"normalize_embeddings": True},
)

vector = embeddings.embed_query("What is LangChain?")
print(len(vector))   # 384

Measuring embedding similarity

import numpy as np

def cosine_similarity(a: list, b: list) -> float:
    a, b = np.array(a), np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

v1 = embeddings.embed_query("Docker runs containers")
v2 = embeddings.embed_query("Containers are created by Docker")
v3 = embeddings.embed_query("The weather is sunny today")

print(cosine_similarity(v1, v2))   # ~0.95 — very similar
print(cosine_similarity(v1, v3))   # ~0.12 — not related

Caching embeddings

from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore

store = LocalFileStore("./embedding_cache/")

# Wrap any embeddings model with a cache
# Identical texts are only embedded ONCE — great for re-indexing
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    underlying_embeddings=OpenAIEmbeddings(model="text-embedding-3-small"),
    document_embedding_cache=store,
    namespace="openai-small-v1",
)

# First call: hits OpenAI API
vectors1 = cached_embeddings.embed_documents(texts)

# Second call: reads from disk cache instantly
vectors2 = cached_embeddings.embed_documents(texts)
Embedding model must stay consistent. You MUST use the same embedding model at both indexing time and query time — mixing models will give garbage results since the vector spaces are incompatible. If you change models, re-index all your documents. For most production workloads, text-embedding-3-small with dimension reduction to 512 gives excellent results at low cost.
