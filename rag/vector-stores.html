Vector Stores

A vector store persists embeddings and enables fast similarity search. It stores your document chunks alongside their vectors and lets you query with "find the K most similar chunks to this question." Choosing the right store depends on scale, persistence needs, and infrastructure.

Vector store comparison

StorePersistenceScaleBest for
ChromaLocal file / in-memoryMillionsDev, small-medium prod
FAISSIn-memory (save to file)BillionsHigh-performance offline search
PineconeManaged cloudBillionsZero-ops production at scale
pgvectorPostgreSQL extensionMillionsExisting Postgres infrastructure
QdrantSelf-hosted / cloudBillionsFiltering, payload search
WeaviateSelf-hosted / cloudBillionsGraphQL API, multi-modal

Chroma — development to production

pip install langchain-chroma
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# ── Build from documents ──
docs   = PyPDFLoader("docs.pdf").load()
chunks = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200).split_documents(docs)

db = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db",   # omit for in-memory
    collection_name="my_docs",
)

# ── Load existing store ──
db = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embeddings,
    collection_name="my_docs",
)

# ── Search ──
results = db.similarity_search("How does Docker networking work?", k=4)
for doc in results:
    print(doc.page_content[:200])
    print(doc.metadata)

# With relevance scores (0–1, higher = more similar)
results_scored = db.similarity_search_with_relevance_scores("Docker networking", k=4)
for doc, score in results_scored:
    print(f"Score: {score:.3f} | {doc.page_content[:80]}")

# ── Add / delete documents ──
db.add_documents(new_chunks)
db.delete(ids=["doc_id_1", "doc_id_2"])

FAISS — high-performance local search

pip install faiss-cpu   # or faiss-gpu for NVIDIA GPUs
from langchain_community.vectorstores import FAISS

# Build FAISS index
db = FAISS.from_documents(chunks, embeddings)

# Save and reload
db.save_local("faiss_index")
db = FAISS.load_local(
    "faiss_index", embeddings,
    allow_dangerous_deserialization=True,
)

# Merge two FAISS indices
db1 = FAISS.from_documents(docs_a, embeddings)
db2 = FAISS.from_documents(docs_b, embeddings)
db1.merge_from(db2)   # db1 now contains everything

Pinecone — managed cloud vector store

pip install pinecone-client langchain-pinecone
from pinecone import Pinecone, ServerlessSpec
from langchain_pinecone import PineconeVectorStore

pc = Pinecone(api_key="your-api-key")

# Create index (once)
pc.create_index(
    name="my-rag-index",
    dimension=1536,   # match your embedding model
    metric="cosine",
    spec=ServerlessSpec(cloud="aws", region="us-east-1"),
)

# Index documents
db = PineconeVectorStore.from_documents(
    chunks,
    embeddings,
    index_name="my-rag-index",
)

# Query
results = db.similarity_search("What is Kubernetes?", k=4)

pgvector — vectors in PostgreSQL

pip install pgvector psycopg2-binary langchain-postgres
from langchain_postgres import PGVector

CONNECTION = "postgresql+psycopg://user:password@localhost:5432/vectordb"

db = PGVector.from_documents(
    documents=chunks,
    embedding=embeddings,
    connection=CONNECTION,
    collection_name="my_documents",
    pre_delete_collection=False,   # True to wipe and rebuild
)

# Re-open existing collection
db = PGVector(
    embeddings=embeddings,
    collection_name="my_documents",
    connection=CONNECTION,
)

results = db.similarity_search("Docker volumes", k=3)

Metadata filtering

# All major vector stores support metadata filtering
# Chroma example
results = db.similarity_search(
    "vacation policy",
    k=4,
    filter={"department": "HR", "year": 2025},
)

# Pinecone filter syntax
results = db.similarity_search(
    "security requirements",
    k=4,
    filter={"source": {"$in": ["policy.pdf", "handbook.pdf"]}},
)

# This lets you scope retrieval to a user's documents, a date range, etc.
Production choice guide: Use Chroma for prototypes and small deployments (<1M vectors). Use FAISS when you need maximum query speed on a single server. Use pgvector if you already run PostgreSQL — zero extra infrastructure. Use Pinecone or Qdrant for large-scale multi-tenant production where you want managed hosting and advanced filtering.
