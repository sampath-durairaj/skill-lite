Text Splitting

Text splitting (chunking) breaks long documents into pieces small enough to embed and retrieve efficiently. The size and overlap of chunks directly affects retrieval quality — too small loses context, too large reduces precision and wastes tokens.

Splitter comparison

SplitterBest forKey parameter
RecursiveCharacterTextSplitterGeneral text (most common)chunk_size, chunk_overlap
CharacterTextSplitterSimple split on one separatorseparator
MarkdownHeaderTextSplitterMarkdown files, preserves structureheaders_to_split_on
RecursiveJsonSplitterJSON datamax_chunk_size
Language (code splitter)Source code fileslanguage enum
SemanticChunkerSemantic coherence (LLM-based)breakpoint_threshold_type

RecursiveCharacterTextSplitter — the default choice

from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,      # target size in characters
    chunk_overlap=200,    # overlap between consecutive chunks
    length_function=len,  # how to measure size (len = chars)
    add_start_index=True, # add char offset to metadata
)

# Split raw text
text = "Long document text here..."
chunks = splitter.split_text(text)
print(f"{len(chunks)} chunks, first: {chunks[0][:80]}")

# Split Documents (preserves metadata)
from langchain_community.document_loaders import PyPDFLoader
docs   = PyPDFLoader("report.pdf").load()
chunks = splitter.split_documents(docs)
print(chunks[0].metadata)  # {"source": "report.pdf", "page": 0, "start_index": 0}

How recursive splitting works

The splitter tries each separator in order until chunks fit within chunk_size:

# Default separators tried in order:
separators = [
    "\n\n",   # paragraph break  ← tried first
    "\n",     # line break
    " ",      # word break
    "",       # character break  ← last resort
]

# Custom separators for a specific format
splitter = RecursiveCharacterTextSplitter(
    chunk_size=800,
    chunk_overlap=100,
    separators=["\n## ", "\n### ", "\n\n", "\n", " ", ""],
)

Token-based splitting (for LLM token limits)

pip install tiktoken
from langchain_text_splitters import RecursiveCharacterTextSplitter
import tiktoken

# Split by tokens instead of characters
enc = tiktoken.get_encoding("cl100k_base")  # OpenAI's tokenizer

splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
    model_name="gpt-4o",
    chunk_size=500,       # 500 tokens per chunk
    chunk_overlap=50,
)
chunks = splitter.split_documents(docs)

# Or with HuggingFace tokenizer
from langchain_text_splitters import RecursiveCharacterTextSplitter
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("sentence-transformers/all-MiniLM-L6-v2")
splitter  = RecursiveCharacterTextSplitter.from_huggingface_tokenizer(
    tokenizer, chunk_size=256, chunk_overlap=32
)

MarkdownHeaderTextSplitter — structure-aware

from langchain_text_splitters import MarkdownHeaderTextSplitter, RecursiveCharacterTextSplitter

markdown = """
# Docker Guide

## Introduction
Docker is a platform for containerisation.

## Installation
### Linux
Run the install script.

### Windows
Use Docker Desktop.
"""

md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=[
        ("#",  "h1"),
        ("##", "h2"),
        ("###","h3"),
    ],
    strip_headers=False,
)
splits = md_splitter.split_text(markdown)
# Each split has header context in metadata:
# {"h1": "Docker Guide", "h2": "Installation", "h3": "Linux"}

# Then further split by size
char_splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
final_chunks  = char_splitter.split_documents(splits)

Code splitter — language-aware

from langchain_text_splitters import Language, RecursiveCharacterTextSplitter

# Split Python code — respects class/function boundaries
python_splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=1000,
    chunk_overlap=100,
)

with open("app.py") as f:
    code = f.read()

chunks = python_splitter.split_text(code)

# Supported languages:
# Language.PYTHON, Language.JS, Language.TS, Language.JAVA,
# Language.GO, Language.RUST, Language.C, Language.CPP,
# Language.HTML, Language.MARKDOWN, Language.SQL

SemanticChunker — meaning-based boundaries

pip install langchain-experimental
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

# Split where semantic meaning shifts significantly
splitter = SemanticChunker(
    embeddings=OpenAIEmbeddings(),
    breakpoint_threshold_type="percentile",   # or "standard_deviation", "interquartile"
    breakpoint_threshold_amount=95,
)
chunks = splitter.split_documents(docs)

Chunk size guide

Content typeRecommended chunk sizeOverlap
General prose (articles, docs)800–1200 chars150–200
Technical docs with code500–800 chars100
FAQs / short Q&A200–400 chars50
Legal / dense text1000–1500 chars200–300
Source code500–1000 tokens50–100
Always inspect your chunks before indexing. Print the first 5, check for broken sentences or code that got split mid-function. The right chunk size is a function of your embedding model's context window, the precision your retriever needs, and the LLM's context limit. Start at 1000/200 and tune from there.
