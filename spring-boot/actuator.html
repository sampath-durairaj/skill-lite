Actuator

Spring Boot Actuator adds production-ready features to your application: health checks, metrics, thread dumps, environment info, and more — all exposed as HTTP endpoints. Kubernetes liveness and readiness probes, Prometheus scraping, and on-call dashboards all integrate directly with Actuator.

Dependency

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

<!-- Expose metrics to Prometheus -->
<dependency>
  <groupId>io.micrometer</groupId>
  <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>

Built-in endpoints

EndpointPathWhat it shows
health/actuator/healthApplication and component health status
info/actuator/infoArbitrary app info (version, git commit, etc.)
metrics/actuator/metricsList of available metrics
metrics/{name}/actuator/metrics/jvm.memory.usedSpecific metric details and tags
prometheus/actuator/prometheusAll metrics in Prometheus text format
env/actuator/envEnvironment properties (sanitized)
loggers/actuator/loggersCurrent log levels; can change at runtime
threaddump/actuator/threaddumpJVM thread dump
heapdump/actuator/heapdumpJVM heap dump file
beans/actuator/beansAll Spring beans in the context
mappings/actuator/mappingsAll @RequestMapping routes
shutdown/actuator/shutdownGraceful shutdown (POST, disabled by default)

Configuration

management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus  # whitelist
        # include: "*"                           # expose all (not for public)
      base-path: /actuator                       # default; change if needed
  endpoint:
    health:
      show-details: when-authorized              # never | always | when-authorized
      show-components: always
    prometheus:
      enabled: true
  info:
    env:
      enabled: true                              # expose info.* properties in /info
    git:
      mode: full                                 # include git.properties in /info

# App metadata shown in /info
info:
  app:
    name: my-service
    version: 1.4.2
    description: Product catalogue API

Health indicators

Spring Boot auto-configures health indicators for common components. Each shows UP, DOWN, or OUT_OF_SERVICE.

# GET /actuator/health
{
  "status": "UP",
  "components": {
    "db": { "status": "UP", "details": { "database": "PostgreSQL", "validationQuery": "isValid()" } },
    "diskSpace": { "status": "UP", "details": { "free": 128849018880, "threshold": 10485760 } },
    "redis": { "status": "UP" }
  }
}

Custom health indicator

@Component
public class PaymentGatewayHealthIndicator implements HealthIndicator {

    private final PaymentClient client;

    @Override
    public Health health() {
        try {
            boolean reachable = client.ping();
            return reachable
                ? Health.up().withDetail("provider", "Stripe").build()
                : Health.down().withDetail("reason", "Ping failed").build();
        } catch (Exception e) {
            return Health.down(e).build();
        }
    }
}

Kubernetes probes

Spring Boot auto-configures /actuator/health/liveness and /actuator/health/readiness when running in Kubernetes (detected from env vars).

# Kubernetes deployment manifest
livenessProbe:
  httpGet:
    path: /actuator/health/liveness
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10

readinessProbe:
  httpGet:
    path: /actuator/health/readiness
    port: 8080
  initialDelaySeconds: 10
  periodSeconds: 5

Custom metrics with Micrometer

@Service
public class OrderService {

    private final Counter ordersCreated;
    private final Timer orderProcessingTime;

    public OrderService(MeterRegistry registry) {
        this.ordersCreated = registry.counter("orders.created",
                "currency", "USD");
        this.orderProcessingTime = registry.timer("orders.processing.time");
    }

    public Order placeOrder(OrderRequest req) {
        return orderProcessingTime.record(() -> {
            Order order = processOrder(req);
            ordersCreated.increment();
            return order;
        });
    }
}

Changing log level at runtime

# View current level for a package
curl http://localhost:8080/actuator/loggers/com.example.myapp

# Set to DEBUG at runtime (no restart needed)
curl -X POST http://localhost:8080/actuator/loggers/com.example.myapp \
  -H 'Content-Type: application/json' \
  -d '{"configuredLevel": "DEBUG"}'
Security: never expose Actuator endpoints to the public internet, especially /env, /heapdump, and /shutdown. In production, run Actuator on a separate management port accessible only internally, or protect endpoints with Spring Security.
