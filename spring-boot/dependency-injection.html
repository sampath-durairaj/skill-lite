Dependency Injection

Dependency Injection (DI) is the foundation of Spring. Instead of classes creating their own dependencies, Spring's IoC (Inversion of Control) container creates and injects them. This makes code modular, testable, and easy to swap out implementations.

Stereotype annotations

Spring discovers beans by scanning for these annotations in your application package and all sub-packages.

AnnotationRoleLayer
@ComponentGeneric Spring-managed beanAny
@ServiceBusiness logicService layer
@RepositoryData access; also translates DB exceptionsPersistence layer
@ControllerHandles HTTP requests (MVC)Web layer
@RestController@Controller + @ResponseBodyWeb layer (REST)

Constructor injection (recommended)

@Service
public class OrderService {

    private final OrderRepository orderRepo;
    private final PaymentService paymentService;
    private final EmailService emailService;

    // Spring injects all three — no @Autowired needed with a single constructor
    public OrderService(OrderRepository orderRepo,
                        PaymentService paymentService,
                        EmailService emailService) {
        this.orderRepo      = orderRepo;
        this.paymentService = paymentService;
        this.emailService   = emailService;
    }

    public Order placeOrder(OrderRequest req) {
        Order order = orderRepo.save(new Order(req));
        paymentService.charge(order);
        emailService.sendConfirmation(order);
        return order;
    }
}
Why constructor injection? Dependencies are explicit (visible in the constructor signature), the object is always fully initialised, fields can be final for thread safety, and it is trivially unit-testable without a Spring context.

Field injection (avoid in production code)

@Service
public class BadExample {
    @Autowired
    private UserRepository repo;  // hidden dependency, can't be final, hard to test
}

@Bean factory methods

Use @Bean in a @Configuration class to register third-party objects or objects that need construction logic.

@Configuration
public class AppConfig {

    @Bean
    public ObjectMapper objectMapper() {
        return JsonMapper.builder()
                .addModule(new JavaTimeModule())
                .disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS)
                .build();
    }

    @Bean
    public RestTemplate restTemplate(RestTemplateBuilder builder) {
        return builder
                .setConnectTimeout(Duration.ofSeconds(5))
                .setReadTimeout(Duration.ofSeconds(10))
                .build();
    }
}

Conditional beans

// Only create this bean when a property is set
@Bean
@ConditionalOnProperty(name = "feature.email.enabled", havingValue = "true")
public EmailService emailService() {
    return new SmtpEmailService();
}

// Provide a fallback if no other EmailService bean is defined
@Bean
@ConditionalOnMissingBean(EmailService.class)
public EmailService noopEmailService() {
    return new NoopEmailService();
}

// Only active on specific profiles
@Bean
@Profile("dev")
public DataSource h2DataSource() { ... }

@Bean
@Profile("prod")
public DataSource postgresDataSource() { ... }

Qualifiers — disambiguating multiple beans

@Bean("primaryCache")
public CacheManager redisCacheManager() { ... }

@Bean("localCache")
public CacheManager caffeineCacheManager() { ... }

// Inject by name
@Service
public class ProductService {
    public ProductService(@Qualifier("primaryCache") CacheManager cache) {
        this.cache = cache;
    }
}

Lifecycle callbacks

@Component
public class ConnectionPool {

    @PostConstruct          // called after injection, before the bean is used
    public void init() {
        // open connections
    }

    @PreDestroy             // called on context shutdown
    public void cleanup() {
        // close connections
    }
}
