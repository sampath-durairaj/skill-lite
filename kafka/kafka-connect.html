Kafka Connect

Kafka Connect is a scalable, fault-tolerant framework for streaming data between Kafka and external systems — databases, object stores, search indexes, and more — without writing a single line of producer or consumer code.

Source vs Sink connectors

TypeDirectionExamples
Source connectorExternal system → Kafka topicDebezium (database CDC), File Source, JDBC Source, S3 Source
Sink connectorKafka topic → External systemJDBC Sink, S3 Sink, Elasticsearch Sink, BigQuery Sink

Standalone vs distributed mode

ModeWhen to useOffset storage
StandaloneDevelopment, single-node pipelinesLocal file
DistributedProduction — multiple workers, fault-tolerantKafka internal topics

In distributed mode, worker configuration (connector definitions, offsets, status) is stored in three internal Kafka topics: connect-configs, connect-offsets, and connect-status.

Starting a Connect worker

# Distributed worker config (connect-distributed.properties)
bootstrap.servers=localhost:9092
group.id=connect-cluster

key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
key.converter.schemas.enable=false
value.converter.schemas.enable=false

offset.storage.topic=connect-offsets
offset.storage.replication.factor=1
config.storage.topic=connect-configs
config.storage.replication.factor=1
status.storage.topic=connect-status
status.storage.replication.factor=1

plugin.path=/opt/kafka/plugins

# Start the worker
connect-distributed.sh connect-distributed.properties

Deploying connectors via the REST API

Connect exposes a REST API on port 8083 (default). All connector lifecycle operations go through it.

# List all available connector plugins
curl http://localhost:8083/connector-plugins | jq .

# Create a JDBC Source connector (polls a database table)
curl -X POST http://localhost:8083/connectors \
  -H "Content-Type: application/json" \
  -d '{
    "name": "orders-jdbc-source",
    "config": {
      "connector.class": "io.confluent.connect.jdbc.JdbcSourceConnector",
      "connection.url": "jdbc:postgresql://db:5432/shop",
      "connection.user": "kafka",
      "connection.password": "secret",
      "table.whitelist": "orders",
      "mode": "incrementing",
      "incrementing.column.name": "id",
      "topic.prefix": "db-",
      "poll.interval.ms": "1000"
    }
  }'

# Check connector status
curl http://localhost:8083/connectors/orders-jdbc-source/status | jq .

# Pause / resume
curl -X PUT http://localhost:8083/connectors/orders-jdbc-source/pause
curl -X PUT http://localhost:8083/connectors/orders-jdbc-source/resume

# Delete connector
curl -X DELETE http://localhost:8083/connectors/orders-jdbc-source

Debezium — Change Data Capture

Debezium is the most widely used Kafka Connect source connector. It reads the database's transaction log (replication slot for PostgreSQL, binlog for MySQL) and emits one Kafka event per row change — no polling, no missed updates.

curl -X POST http://localhost:8083/connectors \
  -H "Content-Type: application/json" \
  -d '{
    "name": "postgres-cdc",
    "config": {
      "connector.class": "io.debezium.connector.postgresql.PostgresConnector",
      "database.hostname": "postgres",
      "database.port": "5432",
      "database.user": "replicator",
      "database.password": "secret",
      "database.dbname": "shop",
      "database.server.name": "shop",
      "plugin.name": "pgoutput",
      "table.include.list": "public.orders,public.products",
      "slot.name": "debezium_slot",
      "publication.name": "debezium_pub"
    }
  }'

Each row change produces an event on topic shop.public.orders with a before/after envelope:

{
  "before": null,
  "after": { "id": 1, "customer_id": 42, "total": 99.99, "status": "placed" },
  "op": "c",
  "ts_ms": 1720000000000
}

Single Message Transforms (SMTs)

SMTs let you modify records in-flight without custom code. They chain together and apply to every record the connector produces or consumes.

{
  "transforms": "unwrap,addTimestamp",
  "transforms.unwrap.type": "io.debezium.transforms.ExtractNewRecordState",
  "transforms.unwrap.drop.tombstones": "false",
  "transforms.addTimestamp.type": "org.apache.kafka.connect.transforms.InsertField$Value",
  "transforms.addTimestamp.timestamp.field": "ingest_ts"
}
